# BYStrike Web AR Demo

Это веб-демо для GitHub Pages. После нажатия **«Стартовать матч»** страница запрашивает разрешение на камеру и показывает живое видео с телефона, поверх которого отображается интерфейс BYStrike.

## Что умеет

- главное меню
- лобби с игроками
- отсчёт 3 → 2 → 1 → GO
- автоматический запрос доступа к камере
- задняя камера телефона при наличии
- HUD поверх живого видео
- цели, lock-on, выстрелы, урон, перезарядка
- экран результатов

## Как загрузить на GitHub Pages

1. Создай новый репозиторий на GitHub.
2. Загрузи в него файлы:
   - `index.html`
   - `styles.css`
   - `script.js`
   - `README.md`
3. Открой `Settings` → `Pages`.
4. В `Build and deployment` выбери:
   - **Source** → `Deploy from a branch`
   - **Branch** → `main` и `/root`
5. Сохрани настройки.
6. GitHub даст ссылку вида:
   `https://yourname.github.io/repository-name/`

## Важно

Камера в браузере обычно работает только по **HTTPS** или на `localhost`.
Поэтому для телефона используй именно GitHub Pages, а не просто открытие файла из памяти телефона.

## Как показывать инвесторам

1. Открой ссылку с телефона.
2. Нажми `Создать матч`.
3. Нажми `Стартовать матч`.
4. Разреши доступ к камере.
5. Появится живое изображение с камеры и поверх него HUD шутера.

## Ограничения

Это презентационное демо, а не настоящая многопользовательская AR-игра. Здесь нет реального GPS-трекинга, сетевого матча и распознавания игроков. Но визуально это уже выглядит как рабочий игровой прототип для питча.
